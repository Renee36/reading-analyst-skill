# 阅读分析师 Skill

## 输入要求

接受 Excel、CSV、文本列表或手动列举的阅读记录。

关键字段（尽量获取）：

| 字段 | 必要性 |
|------|--------|
| 书名 | **必须** |
| 年份 | 强烈建议 |
| 分类 | 强烈建议 |
| 完成状态 | 建议 |
| 评分 | 建议 |
| 点评/笔记 | 可选（有则能生成更深洞察） |

## 数据处理流程

复制此清单追踪进度：

```
数据处理进度：
- [ ] 第一步：数据提取与清洗
- [ ] 第二步：统计分析
- [ ] 第三步：内部一致性预检
- [ ] 第四步：生成十章报告
- [ ] 第五步：质量验证
```

### 第一步：数据提取与清洗
1. 读取所有 Sheet / 数据源，逐 Sheet 适配列结构
2. 提取有效书目条目（书名字段必须存在）
3. **去重处理**：同一本书跨多年 → 以最早标记「已完成」的年份为准
4. 区分已完成 vs 未完成书目

### 第二步：统计分析
计算：总已读数量（去重后）、年度阅读量分布、品类分布（数量 + 占比）、评分统计（有评分书目数、均分、最高分、最低分）、各品类平均评分。

### 第三步：内部一致性预检

**在生成任何章节之前，先构建以下索引** —— 这能避免最常见的错误（跨章节数据矛盾）：

- **高分书索引**：所有评分 ≥4.8 的书目，含书名、评分、品类、年份 → 用于第三章和第四章
- **年度统计索引**：每年已读本数 → 用于第一章和第五章
- **品类-维度映射表**：每本书 → 30 维度中的某一个 → 用于第六章

### 第四步：生成十章报告

按以下结构输出。第六章知识维度参考资料见 [DIMENSIONS_CN.md](DIMENSIONS_CN.md)。

### 第五步：质量验证

交付前执行本文件末尾的质量检查清单。

## 报告结构（十章）

### 第一章：总览数据
- 时间跨度、已读总量、有评分书目数、平均分、涉及品类数
- **年度阅读量走势**：用 `█░` 柱状图可视化，附一句趋势概括
- 示例：`2024 ██████████████████████████████████░ 34`

### 第二章：阅读品类分布
- 表格：品类 | 数量 | 占比 | 说明 —— 按数量降序排列，占比之和 ≈100%

### 第三章：品味画像
- **一句话人格** + **6-10 个人物刻画关键词**（行内代码格式，如 `方法论控` `长期主义`）
- 3-5 个品味特征，每个含：特征名称、具体书名证据、评分佐证（**必须与第四章完全一致**）、品味判断
- ⚠ 引用任何评分数据前，先查阅高分书索引

### 第四章：满分及高分书目
- 15-25 本最高评分书目表格（≥4.8 分）：评分 | 书名 | 品类 | 年份 —— 按评分降序
- 2-3 段总结：年份聚集模式、高分书共同特质、读者最看重什么

### 第五章：阅读演化脉络
- 3-6 个阅读阶段表格：阶段名 | 年份 | 关键词 | 特征描述
- 2-3 段总结：兴趣迁移路径、转折点、预测下一阶段

### 第六章：知识网络全景评估

**核心章节——必须全面覆盖人类知识版图，不局限于用户已读品类。**

采用 8 大领域 × 30 维度框架。完整维度清单、分类逻辑和评级标准见 [DIMENSIONS_CN.md](DIMENSIONS_CN.md)。

必须包含的子章节：
- **A. 详评**：8 大领域分组表格，组内维度按评级降序，每个维度必须有代表书目或"无"
- **B. 雷达图**：`█░` 柱状图展示全部 30 维度（1-10 分），按领域分组，标注极端值（`← 绝对优势`、`← 空白`）
- **C. 缺陷总览**：所有空白领域（1/10）表格，含"为什么重要"（关联读者生活场景）+ 入门推荐
- **D. 小结**：知识结构形态标签（如「倒T型」）、30 维度统计（优势区/有基础/薄弱各几个）、优先填补的 3 大方向

### 第七章：阅读推荐

**书单 A：填补空白（10 本）**：
- 覆盖第六章所有空白领域（1 分），设计阅读路径
- 每本书：为什么推荐这本而不是其他，关联读者品味
- **确认用户未读过**

**书单 B：深耕热爱（10 本）**：
- 基于读者评分最高的 3-5 个领域
- 每本书必须说明与已读高分书的衔接（如"读了《XX》★5 之后，这本从更深层面解释……"）
- **确认用户未读过**

### 第八章：有趣的洞察
- 5-7 个数据隐藏发现（执念书单、转折点、计划 vs 实际、文化暗线、数量跃迁等）
- 每个洞察必须有**具体书名和数据**支撑

### 第九章：阅读的开创价值
- **9.1 独特知识组合**：跨领域交叉点用 `A × B × C` 格式展示，稀缺性分析
- **9.2 输出方向**：3-5 个具体提案（方向名称、为什么你能做、目标受众、实现形式）
- **9.3 如何帮到更多人**：可复制的方法、最佳分享切入点、目标人群——关联职业和生活角色

### 第十章：附录（可选）
- 按年份完整书目清单。如 >200 本，建议单独导出。

## 输出要求

- **格式**：Markdown 文件（`.md`）
- **语言**：匹配输入数据语言
- **保存位置**：询问用户，默认桌面
- **文件名**：`{用户名}阅读分析报告_{起始年份}-{结束年份}.md`

## 质量检查清单

```
质量验证：
- [ ] 书目已去重，无重复计数
- [ ] 品类占比之和 ≈100%
- [ ] 第三章评分引用与第四章表格完全一致（最常见错误！）
- [ ] 第一章年度数据与第五章脉络一致
- [ ] 第六章雷达图评分与详评表格一致
- [ ] 第六章覆盖全部 30 维度，每个领域内按评级降序排列
- [ ] 第六章所有空白领域已识别并有入门推荐
- [ ] 书单 A 覆盖所有 1 分空白领域
- [ ] 书单 B 每本书与已读高分书有明确衔接
- [ ] 推荐的 20 本书确认用户均未读过
- [ ] 第三章有 6-10 个人物关键词
- [ ] 第八章每个洞察有具体书名和数据
- [ ] 第九章输出方向具体可行
```

## 核心原则

1. **全面性**：知识评估覆盖人类知识全版图，空白区和优势区同等重要
2. **数据驱动**：所有结论有书目数据支撑，不做无依据的猜测
3. **内部一致性**：生成前先构建统一数据索引，各章节之间不能前后矛盾
4. **个性化**：品味画像和洞察针对这位具体读者，不套用模板话术
5. **可行动**：双轨推荐，每本书有明确的推荐逻辑
6. **诚实评价**：不回避空白区，如实标记 1 分维度
7. **面向未来**：不仅分析过去，还要回答"这些阅读能创造什么价值"

## 异常处理

- **书目 <10 本**：提醒用户，跳过第五章和第八章
- **无评分**：跳过第四章，第三章仅基于品类模式生成，注明限制
- **无年份**：跳过第五章，第一章仅展示总量无走势图，注明限制
- **格式不可识别**：请用户澄清，提供格式示例
- **去重不确定**（不同版本/译名）：列出存疑条目，请用户确认
- **>500 本**：先生成 Ch1-Ch9，第十章按需单独生成
